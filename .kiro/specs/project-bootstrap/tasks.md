# Implementation Plan: Project Bootstrap

## Overview

Scaffold a buildable Next.js App Router project with TypeScript strict mode, Tailwind CSS, ESLint, and Prettier. Create the full directory skeleton from `steering/structure.md`. No application logic — just the foundation for all future specs.

## Tasks

- [x] 1. Scaffold Next.js project
  - [x] 1.1 Run `pnpm create next-app` with flags: TypeScript, Tailwind, ESLint, App Router, `src/` directory, `@/*` import alias, pnpm
    - Run from the project root, targeting the current directory
    - The scaffolder determines exact versions of Next.js, Tailwind, and ESLint config format
    - Preserve existing files: `context/`, `.kiro/`, `CLAUDE.md`
    - _Requirements: R1.1, R1.2, R1.3, R2.1, R3.1, R4.1_

  - [x] 1.2 Verify scaffolded configuration
    - Confirm `tsconfig.json` has `"strict": true` under `compilerOptions`
    - Confirm `tsconfig.json` has `@/*` path mapping to `./src/*`
    - Confirm ESLint config includes `eslint-config-next` (framework-recommended ruleset)
    - Confirm `pnpm-lock.yaml` exists
    - _Requirements: R1.3, R1.4, R3.1, R4.1, R4.2_

- [x] 2. Configure Prettier and format scripts
  - [x] 2.1 Install Prettier and create configuration
    - Add `prettier` as a dev dependency: `pnpm add -D prettier`
    - Create `.prettierrc` at project root with project conventions (semicolons, single quotes, 2-space tabs, trailing commas, print width)
    - _Requirements: R3.2_

  - [x] 2.2 Add format scripts to `package.json`
    - Add `"format": "prettier --write \"src/**/*.{ts,tsx,css,md,mdx}\" \"content/**/*.{md,mdx,yaml,yml,json}\""` script
    - Add `"format:check": "prettier --check \"src/**/*.{ts,tsx,css,md,mdx}\" \"content/**/*.{md,mdx,yaml,yml,json}\""` script
    - `format` writes changes (developer use), `format:check` exits non-zero if unformatted (CI use)
    - _Requirements: R3.3_

- [x] 3. Configure Tailwind content scanning and simplify starter files
  - [x] 3.1 Add `@source` directive to global stylesheet
    - In `src/styles/globals.css` (or `src/app/globals.css` depending on scaffolder output), add a `@source` directive pointing to `../../content/` so Tailwind scans content files for utility classes
    - Keep the Tailwind `@import "tailwindcss"` directive (or equivalent generated by scaffolder)
    - Remove any scaffolder-generated demo styles, keep only Tailwind imports and the `@source` directive
    - _Requirements: R2.2_

  - [x] 3.2 Simplify root layout (`src/app/layout.tsx`)
    - Replace scaffolder-generated layout with a minimal Server Component
    - Render `<html lang="en">` and `<body>` with `{children}`
    - Export metadata with `title: "Lorenzo Santucci"`
    - Import the global stylesheet so Tailwind classes render on all pages
    - No navigation, footer, font imports, or providers — just the shell
    - _Requirements: R2.3, R6.1, R6.2_

  - [x] 3.3 Simplify placeholder page (`src/app/page.tsx`)
    - Replace scaffolder-generated page with a minimal Server Component
    - Render `<main>` containing `<h1>Lorenzo Santucci</h1>`
    - No styling beyond Tailwind defaults, no images or links
    - _Requirements: R6.3_

- [x] 4. Create directory skeleton with `.gitkeep` files
  - [x] 4.1 Create all skeleton directories and `.gitkeep` files
    - Create each directory listed in `steering/structure.md` that doesn't already exist
    - Add a `.gitkeep` file to each empty directory (no source files):
      - `src/components/.gitkeep`
      - `src/lib/.gitkeep`
      - `src/world/.gitkeep`
      - `content/pages/.gitkeep`
      - `content/projects/.gitkeep`
      - `content/blog/.gitkeep`
      - `content/agents/.gitkeep`
      - `content/locations/.gitkeep`
      - `content/characters/.gitkeep`
      - `content/dialogues/.gitkeep`
      - `content/services/.gitkeep`
      - `public/assets/.gitkeep`
    - Do NOT add `.gitkeep` to directories that already contain source files (`src/app/`, `src/styles/`)
    - Do NOT modify or remove `context/`, `.kiro/`, or `CLAUDE.md`
    - _Requirements: R5.1, R5.2, R5.3_

  - [x] 4.2 Write property test for directory skeleton completeness
    - **Property 1: Directory skeleton completeness**
    - For any directory listed in `steering/structure.md`, that directory SHALL exist on disk, and if it contains no source files, it SHALL contain a `.gitkeep` file
    - Note: No test runner is installed in this spec. Write this as a standalone TypeScript/Node script that can be run with `npx tsx` to verify the skeleton. Convert to a proper test when a test runner is introduced in a later spec.
    - **Validates: R5.1, R5.2**

- [x] 5. Run format and verify
  - [x] 5.1 Run Prettier on the codebase
    - Execute `pnpm format` to format all source and content files to match the Prettier config
    - This ensures the initial codebase passes `format:check` from the start
    - _Requirements: R3.2, R3.4_

- [x] 6. Checkpoint — Verify all gates pass
  - Run `pnpm install` — completes without errors
  - Run `pnpm build` — compiles with zero errors (validates TypeScript strict mode, path alias, Tailwind config, Server Components)
  - Run `pnpm lint` — reports zero errors
  - Run `pnpm format:check` — exits zero (code matches formatter config)
  - Confirm `context/`, `.kiro/`, and `CLAUDE.md` are untouched
  - Ensure all tests pass, ask the user if questions arise.
  - _Requirements: R1.4, R3.4 (verification gates)_

## Notes

- Tasks marked with `*` are optional and can be skipped for faster MVP
- The scaffolder determines exact versions — no hardcoded version numbers in tasks
- ESLint config format (flat vs legacy) is whatever `create-next-app` generates
- Tailwind v4 uses CSS-based config (`@import "tailwindcss"` + `@source`), not `tailwind.config.ts`
- Each task references specific requirements for traceability
