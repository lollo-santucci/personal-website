# Product Context

## What This Is

Personal website for Lorenzo Santucci, freelance full-stack developer and ML/AI engineer. Not a portfolio — a **modular content platform** with two views over a shared content system.

## Two Views, One Content System

- **Classic view**: Professional website (desktop + responsive mobile). Pages, projects, blog, agentdex, office, contact. Modern UI with pixel-art accents.
- **Play view**: 2D pixel-art world (desktop: full world; mobile: game-boy UI). Same content accessed through exploration, characters, dialogue.

Both read from one content system. Adding content is configuration, not code.

## Target Users

1. **Potential clients** — Need clarity, credibility, conversion. Classic view users. The site must work as a professional website first.
2. **Recruiters & partners** — Quick access to skills, projects, profile. Speed matters.
3. **Curious visitors** — Exploration-oriented. Play view users. Need depth, personality, shareability.
4. **Lorenzo (admin)** — Adds content via config, manages agents, extends the system.

## Core Principles

- **Content-first architecture**: Content entities defined before and independently of views
- **No duplication**: World references content, never copies it
- **Classic view stands alone**: 100% complete without play view
- **Modular & extensible**: New content added via config, not code changes
- **No gates**: Play mode is an invitation, never a prerequisite

## Key Content Types

Page, Project, Blog Post, Agent, Location, Character, Dialogue, Service.

## World Locations → Content

Casa → About | Edicola → Blog | Shop → Projects | Ufficio → Office/Agents

## Objectives Priority

1. Client acquisition and conversion
2. Technical credibility through real work
3. Distinctive brand identity
4. Content platform (config-level additions)
5. Agent platform (AI entities with triple nature)
6. Exploration and delight

## Guardrails

- No friction before value — classic view must work perfectly standalone
- No gimmick over substance — every feature serves a purpose
- Fun never undermines competence — humor reinforces skill
- No empty world elements — every location/NPC serves content
- Real content only — no placeholder text at launch
- Performance — the site must be fast
