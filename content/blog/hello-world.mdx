Every developer eventually builds a personal website. Most of us build it three or four times, actually, because the first version is always embarrassing in retrospect. This is my latest attempt, and it's a bit different from the usual portfolio template.

The idea was simple: what if a personal site had two ways to experience the same content? A classic professional view for people who want information fast, and a pixel-art world for people who want to explore. Same content underneath, two completely different interfaces on top.

## The Architecture Rabbit Hole

The interesting part isn't the pixel art — it's the content system. Every piece of content lives in one place: MDX files for long-form text, YAML for structured data. The classic view reads from it. The world reads from it. Neither view owns the content, and neither duplicates it.

This means adding a new project is just creating a file with the right frontmatter. No code changes, no redeployment logic, no CMS. The content loader validates everything at build time with TypeScript types, so if you forget a required field, the build tells you before production does.

## Phaser Meets Next.js

Getting Phaser and Next.js to coexist required some architectural discipline. Phaser owns the canvas and nothing else — no UI, no text overlays, no dialogue boxes. All of that is React with Tailwind, positioned with CSS over the canvas. Zustand handles the communication: Phaser emits events, React responds.

It sounds clean on paper. In practice, it took a few iterations to stop Phaser from trying to own things it shouldn't. The rule is simple: if a user needs to read text or click a button, that's React's job.
